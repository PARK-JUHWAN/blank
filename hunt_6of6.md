# 🎯 6/6 사냥 명세서 (Claude Code 실행용)

> **미션:** osk의 6원칙을 **전부(6/6)** 만족하는 사업 후보를 찾을 때까지 브레인스토밍 + 채점을 반복한다.
> **6/6이 나오는 즉시 응답을 종료한다.** 5/6 이하는 계속 진행.
> **분야 제한 없음.** 우주·심해·법률·게임·온체인 등 어디든. 떠오르는 대로 무차별 투척.

---

## 1. osk의 6원칙 (채점 기준)

각 후보를 아래 6개로 ✅(통과) / △(애매·부분) / ❌(탈락) 채점한다.
**△는 통과로 치지 않는다. 6/6은 6개 전부 ✅일 때만이다.**

| # | 원칙 | ✅ 조건 | ❌ 조건 |
|---|------|--------|--------|
| 1 | **양산 가능** | 클로드코드로 대량 생산. osk는 지휘만 | "잘 만든 하나"여야 함 / 건당 수작업 |
| 2 | **무심판** | 게이트키퍼가 양(量)을 스팸/위반으로 벌하지 않음 | 구글·애플·유튜브·앱마켓·에셋마켓 등 심판자 존재 |
| 3 | **광고/조회수 아님** | 수익원이 사용료·판매·수수료 | 사람 주의(attention)에 의존하는 광고/조회수 |
| 4 | **buyer 지금 존재** | 지금 지갑 충전돼 살 준비된 구매자가 실재 | 미래 수요 / 수요 미성숙 / 안 삼 |
| 5 | **osk 손 거의 안 감** | 발견·영업·설득·운영을 안 해도 굴러감 | 마케팅 필수 / 영업 필요 / 24/7 운영 |
| 6 | **하방 0** | 실패해도 잃는 게 거의 없음 (죽은 산출물 비용 0) | 개발 수개월 / 고정비 / 외부API·실시간 데이터 운영비 |

---

## 2. 채점 시 반드시 적용할 함정 (오판 방지)

- **실시간/갱신 데이터 = 6번 ❌.** 외부 의존·운영 발생. (예: 시세, 날씨, 뉴스 — 전부 탈락)
- **AI가 스스로 할 수 있는 것 = 4번 ❌.** 봇이 직접 하면 안 삼. (예: 번역, 요약, 단순변환, 해시 — buyer 없음)
- **마켓플레이스 경유(Etsy·Gumroad·유니티·앱스토어 등) = 2번 ❌.** 그 마켓이 심판자.
- **오픈소스/무료 배포(npm 등) = 4번 ❌.** 다운로드는 돼도 돈을 안 냄.
- **수익이 "사람이 봐야" 생기면 = 3번 ❌.**
- **buyer가 "있으면 좋은 것" 수준이면 △, "안 사면 일을 못 하는 것"이면 ✅에 가깝게.**

---

## 3. 채점 출력 형식 (후보마다 한 줄)

```
[코드] 후보명 — 1:✅ 2:✅ 3:✅ 4:△ 5:✅ 6:✅ → 5/6 (깨진칸: 4번 buyer 미성숙)
```

- 라운드당 50~100개씩 투척. 짧게. 구구절절 설명 금지.
- 각 라운드 끝에 "이번 라운드 최고 점수 / 6/6 유무"만 한 줄.

---

## 4. 종료 규칙 (중요) — 둘 중 먼저 도달하는 쪽에서 종료

**종료조건 A — 6/6 발견 (성공 종료):**
- 6/6(6개 전부 ✅)이 단 하나라도 나오면 → 그 즉시 모든 작업 중단하고 응답 종료.
- 종료 시 출력: 해당 후보명 + 6원칙 채점표 + "왜 6개가 전부 ✅인지" 각 칸 1줄 근거.

**종료조건 B — 1000개 도달 (소진 종료):**
- 채점한 후보 누적 **1000개**에 도달하면 → 6/6이 없어도 종료. (토큰 폭발 방지)
- 종료 시 출력: "1000개 중 6/6 없음. 최고점 = 5/6." + 5/6 후보 전부 나열(코드·이름·깨진칸만, 한 줄씩).

**진행 중:**
- 위 둘 중 어느 것도 도달 전이면 라운드 계속. 누적 카운터를 라운드마다 표기(예: "누적 350/1000").
- △는 통과 아님. 6/6은 6칸 전부 ✅일 때만.

## 5. 샘플 (채점이 어떻게 도는지 이해용 — 이 후보들을 재사용하라는 뜻 아님)

> 아래는 "채점 방식"을 보여주는 예시일 뿐이다. **새 후보를 직접 생성하라.**

```
[샘플A] x402 순수계산 endpoint(예: 정밀 세무계산)
  1:✅ 양산  2:✅ 무심판(프로토콜)  3:✅ 사용료  4:△ buyer미성숙  5:✅ 자동  6:✅ 운영0
  → 5/6 (깨진칸: 4)

[샘플B] 게임 3D에셋 대량생성 → 유니티 마켓
  1:✅  2:❌ 마켓심판  3:✅  4:✅  5:△ 마케팅  6:✅
  → 4/6 (깨진칸: 2, 5)

[샘플C] 실시간 환율 데이터 판매
  1:✅  2:✅  3:✅  4:✅  5:❌ 운영  6:❌ 외부의존·실시간
  → 4/6 (깨진칸: 5, 6)

[샘플D] 봇용 제재스크리닝/검증 영수증 자판기(x402)
  1:✅  2:✅  3:✅  4:△ 의무성수요 초기  5:✅  6:✅
  → 5/6 (깨진칸: 4)
```

## 6. 사냥 방향 힌트 (막히면 참고 — 강제 아님)

- 6원칙 중 **2번(무심판)과 4번(buyer)이 동시에 ✅인 지점**이 거의 모든 후보의 사망 지점이다. 여기를 뚫는 후보를 우선 탐색.
- "무심판"은 보통 **프로토콜/규격/온체인**에서만 성립. "성숙 buyer"는 보통 **기존 시장**에 있고 거기엔 심판자가 있음. → 이 둘이 만나는 희귀한 교집합을 노려라.
- 분야를 의도적으로 멀리 던져라(우주·해양·농업·물류·에너지·생명과학·법률·교육 등). 가까운 분야는 이미 포화일 확률이 높음.
